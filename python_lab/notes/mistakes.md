## 2026年5月22日: LCS / 不相交线的下标理解

### 易混点

- 在 Uncrossed Lines 中，容易误以为要比较单条线内部的 `i` 和 `j` 大小。
- 实际上 `i` 属于 `nums1`，`j` 属于 `nums2`，它们来自不同数组，`i < j` 或 `i > j` 本身没有判断意义。

### 正确理解

判断两条线是否相交，要比较两条匹配之间的相对顺序。

如果两条线是：

```text
(i1, j1)
(i2, j2)
```

当：

```text
i1 < i2
```

为了不相交，必须有：

```text
j1 < j2
```

如果出现：

```text
i1 < i2
j1 > j2
```

说明两个数组中的匹配顺序反了，线就会相交。

### 下次提醒

- 单条匹配只说明两个值相等。
- 多条匹配要保持两边下标序列都单调递增。
- 这也是 Uncrossed Lines 可以转换成 LCS 的原因。

## 2026年6月8日: 无向图中“环”和“重复边”的区别

### 易混点

在 Union Find 判环题中，容易把“出现环”理解成“出现重复连接”。

### 正确理解

环不是重复边。

重复边是完全相同的边再次出现，例如：

```text
[1, 2]
[1, 2]
```

环是两个已经连通的节点之间又新增了一条边，例如：

```text
[1, 2]
[2, 3]
[1, 3]
```

这里 `[1, 3]` 并不是重复边，但在加入它之前，`1` 和 `3` 已经可以通过 `1 -> 2 -> 3` 连通，所以加入 `[1, 3]` 会形成环。

### 下次提醒

- 判断是否成环，不是看边是否重复。
- 判断是否成环，要看两个端点是否已经属于同一个集合。
- 在并查集中，这个判断就是 `find(a) == find(b)`。
