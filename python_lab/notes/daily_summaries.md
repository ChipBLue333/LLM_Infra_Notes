# 2026年5月2日 训练总结

今天完成了 Phase 1 的前三道基础题目，成功激活了以下核心编程状态和知识点：

### 1. 哈希表 (Hash Map) 与空间换时间 (Problem 001: Two Sum)
- **知识点**：利用 Python 字典 (`dict`) 的 $O(1)$ 查找特性。
- **技巧**：**边历边存 (One-Pass Lookup)**。在遍历数组时，将看过的元素存入字典（值作为 Key，索引作为 Value）。与其向后用双层 `for` 循环寻找配对 ($O(n^2)$)，不如每次计算出需要的“那一半”，直接去字典里以 $O(1)$ 的速度验证，从而将总体时间复杂度优化到 $O(n)$。

### 2. 栈 (Stack) 结构与括号消除 (Problem 002: Valid Parentheses)
- **知识点**：利用 Python 列表 (`list`) 的 `append()` 和 `pop()` 方法来模拟 **后进先出 (LIFO/FILO)** 的栈结构。
- **技巧**：碰到具有“嵌套匹配”性质的问题时，将开头元素压入栈中，遇到结尾元素时弹出栈顶进行配对。
- **工程细节**：
  - 避免使用 Python 关键字（如 `str`, `list`, `dict`）作为变量名。
  - 对于固定的映射关系表（如括号匹配规则 `{"(": ")"}` 等），应当定义在循环外部进行初始化，避免内部重复创建带来性能损耗。

### 3. 双指针 (Two Pointers) 逆向填坑 (Problem 003: Merge Sorted Array)
- **知识点**：使用多个指针在数组上移动以完成查找或修改任务。
- **技巧**：**原地修改 (In-place) 与逆向思维**。当需要原地修改数组、且目标数组尾部带有足够的空余空间时，如果从前往后修改容易覆盖尚未读取的数据。此时采用“从后往前”填坑的策略可以完美避免元素平移，保持 $O(n)$ 时间复杂度和 $O(1)$ 空间复杂度。
